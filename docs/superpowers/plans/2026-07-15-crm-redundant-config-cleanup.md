# CRM 冗余配置清理实施计划

## 目标

在不改动工作台搜索的前提下，完成已确认的 P0、P1、P2 清理：删除旧模型和无人使用的配置，统一重复选项来源，并移除无入口后台页面。

## 实施范围

1. 新增幂等数据库迁移：
   - 将与公共选项集完全一致的字段私有选项迁移为公共选项集引用。
   - 删除无人使用的表单、数据模板、字段、报表用途、规则脚本和财务类型。
   - 删除旧“业务数据”分类和 32 张已无源码模型的历史表。
   - 删除迁移后仍无人使用的公共选项集。
2. 清理后端：
   - 删除旧“业务数据”分类、流程实例数据记录和产品选项集同步分支。
   - 删除已无消费者的报表用途枚举。
3. 清理后台页面：
   - 删除已被嵌入式编辑页替代的部门、数据字段、表单字段列表页及旧资产详情页。
   - 将保留编辑页的父级路由指向真实入口。
4. 验证：
   - 执行迁移并核对配置数量、引用完整性和残留源码引用。
   - 按项目要求，不执行 build 或测试命令。

## 保护措施

- 完整数据库备份：`doc/backups/crm-full-before-p0-p2-cleanup-20260715-031927.dump`
- 所有删除按名称、字段编码和“无任务引用”等条件限定。
- 不修改工作台搜索相关页面、组件或接口。
