# CRM 线索模板设计

## 目标

在线索、客户、资产和流程业务之间建立清晰的数据边界，并允许租户配置线索扩展字段。

## 数据边界

- 线索主表固定保存编号、姓名、手机、微信、来源、渠道、外部线索 ID、状态、查重、负责人和审计字段。
- 新增固定数据模板分类“线索信息”。该分类的扩展值保存在 `Lead.RecordJSON` 的 `data_values` 中，不新增平行数据记录表。
- `InputSnapshotJSON` 继续保存原始录入快照，`RecordJSON` 只保存当前线索扩展字段。
- 线索转客户只复制姓名、手机、微信、来源、渠道和初始诉求；原线索通过 `Lead.CustomerID` 保留，不复制来源扩展字段。
- 客户详情只读展示来源线索主字段及线索扩展模板数据。

## 页面

- 数据模板增加“线索信息”页签。
- 工作台录入线索时，在固定基础字段下按模板分区展示线索扩展字段。
- 后台客户管理增加只读“线索信息”列表，排序在“客户信息”之前。
- 历史“客户来源与基础建档”模板停用，避免城市、初始诉求等信息在线索和客户中重复维护。

## 约束

- 工作流资料模板不允许选择线索信息分类，防止线索字段写入客户数据记录。
- 附件类型暂不在轻量线索录入中开放；后台模板仍可配置，工作台显示为暂不支持。
- 按项目要求不增加或执行自动化测试，仅进行 JSON、格式、残留字段和 Dever 静态审计。
