# CRM 后台客户数据权限与产品展示设计

## 目标

- 线索、客户和资产只能由工作台业务流程创建。
- 后台线索保持当前只读列表；客户和资产可以编辑，但不能新增。
- 后台资产列表去掉独立详情入口。
- 产品列表正确展示产品分类。
- 将产品的“服务流程”改为更准确的“签约后流程”，不改变数据结构和运行逻辑。

## 边界

客户数据的创建必须经过工作台的线索登记、重复检测、转客户和建立资产流程。后台只用于查询与纠错，不提供新增入口，避免绕过这些业务规则。线索状态由工作台操作流转，后台线索列表保持只读。

产品的签约后流程为可选配置：

- 不选择：产品在入口签约流程完成后结束。
- 选择：签约完成后自动进入指定流程，用于租赁等后续运营。

## 实现设计

1. 后台线索列表保持现状，不增加新增、编辑或详情操作。
2. 客户列表删除“新增客户”按钮、新增弹窗和对应状态，保留编辑客户。
3. 资产列表删除“新增资产”按钮、新增弹窗和对应状态，保留编辑资产。
4. 资产列表删除“查看资产详情”操作。详情页文件暂不删除，避免影响可能存在的直接链接；后台菜单和列表不再暴露入口。
5. 产品列表将分类展示路径由错误的 `product_category.name` 改为 Model Relation 实际输出的 `category.name`。
6. 产品列表、编辑表单和校验提示统一使用“签约后流程”文案，底层仍使用 `service_workflow_id`，避免无意义的数据迁移。

## 异常处理

- 后台不提供创建线索、客户和资产的界面入口，现有工作台创建规则不受影响。
- 客户和资产编辑继续使用现有 Provider 校验，不新增平行保存逻辑。
- 签约后流程仍只允许选择已启用的非入口流程，并保留首阶段自动分配校验。

## 验证

只执行 Dever 静态审计、JSON 解析和变更检查。按项目要求，不执行 build 或 test，由用户手动验证管理端创建入口已移除、客户和资产仍可编辑、资产详情入口已移除、产品分类正确展示以及签约后流程可保存。
