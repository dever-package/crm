# 任务能力与字段能力收敛设计

## 目标

- 任务只配置流程行为：预约会议、下次跟进、负责人参会。
- 财务类型和参与统计改为数据字段自身属性，同一字段被多个任务表单复用时只配置一次。
- 不恢复集中式“系统用途”页面，避免字段、用途和任务三层重复映射。
- 历史动态资料、财务流水、统计快照以及旧配置关系全部保留。

## 唯一数据流

- `DataField.finance_type_id`：该字段提交非空金额时写入对应财务类型的 `FinanceLedger`。
- `DataField.stat_enabled`：该字段提交时同步最新值到 `StatFieldValue`。
- 表单任务和审批任务都读取实际提交字段的上述属性，不再读取 `TaskFinanceType` 或 `TaskStatField`。
- 财务流水继续使用 `field:<data_field_id>` 作为稳定来源键，统计快照继续保留来源任务和操作记录。
- 任务关系表与旧“系统用途”表仅保留作历史审计和迁移依据，不参与页面或运行时。

## 页面

- 任务弹窗继续使用“基础信息 / 流程控制 / 业务能力”三个 Tab。
- 业务能力只显示预约会议、下次跟进和负责人参会。
- 数据模板字段编辑弹窗增加“财务类型”和“参与统计”。财务类型为空表示普通字段；参与统计默认关闭。
- 分组子字段使用相同配置，分组和附件本身不参与财务或统计。

## 迁移

- 从历史财务流水和旧财务用途映射回填 `finance_type_id`，优先使用已产生流水的真实字段映射。
- 从历史统计快照、旧统计用途和任务统计关系回填 `stat_enabled`。
- 重新启用此前被任务生成财务控件替代而停用的表单字段。
- 不删除或改写已有 `DataRecord`、`FinanceLedger`、`StatFieldValue`、旧用途表和旧任务关系表。

## 验证约束

项目禁止执行 build 和 test。只做代码格式化、JSON/SQL 静态检查、残留引用检查和 `git diff --check`，运行效果由用户手动验证。
