# CRM 前台 Body 风格改造设计

## 目标

仅改造 CRM `work` 站点的视觉样式，使工作台、线索池、客户列表、详情抽屉和任务弹窗统一采用 `bot/body` 的紧凑工作台风格。后台管理端、API、Service、业务数据和流程行为保持不变。

## 设计选择

在 CRM front plugin 内增加一个仅对 `work` 主框架生效的皮肤节点，由 `front/page/work/main.json` 挂载一次。样式通过 CRM 根作用域约束，不修改 `package/front` 全局主题，也不复制 `bot/body` 的导航和路由实现。

## 视觉令牌

- 字体：`-apple-system, BlinkMacSystemFont, "Segoe UI", "Helvetica Neue", Arial, "Noto Sans SC", "PingFang SC", sans-serif`
- 基础字号：`12.8px`
- 页面背景：`#f4f6f5`
- 内容表面：`#ffffff`
- 主文字：`#171a19`
- 次文字：`#6b7370`
- 弱文字：`#9ca3a0`
- 边框：`#e4e8e6`
- 强边框：`#d2d9d6`
- 激活背景：`#e4e8e6`
- 主色：`#1a4a35`
- 常规圆角：`6px`，紧凑内容圆角：`3px`
- 桌面侧栏：`240px`
- 桌面顶栏：`38px`

## 组件边界

1. 主框架：统一侧栏、顶栏、内容框架、折叠状态和移动端导航。
2. 页面内容：统一页面内边距、标题、说明文字和空状态。
3. 数据界面：统一筛选器、表格、状态标签、分页和操作按钮的字号与密度。
4. 反馈界面：统一抽屉、弹窗、表单控件、任务记录和流程视图。
5. 统计界面：保留现有图表和数据结构，仅调整容器、文字和间距。

## 实现方式

- 新建聚焦的 `work-skin.tsx`，只负责注入作用域样式和管理 CRM work 根标记。
- 在 `plugin.ts` 注册皮肤节点，在 `work/main.json` 挂载一次。
- 给 CRM 工作台页面和关键自定义组件补少量语义 class，避免依赖脆弱的深层 DOM 顺序。
- 复用现有 `Button`、`Input`、`Dialog`、`workApi` 和全部业务组件，不建立第二套 UI 或业务实现。

## 响应式

- 桌面沿用 body 的 240px 侧栏和白色内容框架。
- 中等屏幕压缩侧栏与内容边距。
- 手机端保留现有可用导航行为，表格继续切换为移动列表，控件不得溢出或重叠。

## 验证

- 浏览器分别检查工作台、线索池、客户列表、详情抽屉和任务弹窗。
- 检查桌面与手机视口的字体、间距、溢出、遮挡、控制台错误和失败请求。
- 运行 Dever 静态 audit、JSON 校验和 diff 检查。
- 按项目要求不运行 `npm run build` 或任何测试命令。
